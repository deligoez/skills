# Sample Documents — recipes by document type

How the rules combine for specific legal and professional documents. Each entry is a concrete
recipe drawn from Butterick's before/after examples. Fonts named are illustrative (his own MB Type
families); substitute per `fonts.md`. Numbers reference `page-layout.md`.

## Caption pages

Court captions are governed by strict rules but still leave room for good typography.
- Replace monospaced fonts with a proportional serif (e.g. Equity); replace ALL CAPS with **bold**
  or **small caps**; add letterspacing to court names; remove underlining.
- **Build the caption as a table** — minimum a two-column table for party names and case title;
  expand with rows/columns for attorney and court info. Use cell borders instead of punctuation
  rules (rows of dashes/underscores). Keep any vertical rules thin and set apart from the text.
- Looser line spacing in the top half; reduce oversized first-line indents; separate address lines
  with **hard line breaks**; align line numbers to body-text baselines.
- Slightly increase point size for party names and case title; drop surplus labels ("Telephone",
  "Email") to save space. Apply foreground/background: rules recede behind the text.

## Motions

- Serif body font (Equity), not sans; letterspace ALL-CAPS headings and footer text; remove
  underlining (use em dashes for emphasis, not underscores); one space between sentences; fix
  inconsistent capitalization in subheadings.
- Narrow the line length / widen margins; move body text away from vertical rules; generous white
  space at edges and around headings; reduce footer point size; align line numbers to baselines.
- Enable hyphenation (justified text); use "keep lines together" so headings don't split across
  pages; edit ruthlessly to cut words. Judges deserve careful formatting even under deadline.

## Research memos

Kill the typewriter default (1″ margins, 12 pt, double-spaced — "have you ever seen a book,
newspaper, or magazine that uses this layout? No.").
- Margins **2″ sides, 1.5″ top/bottom**; line length ~**65 characters**; reduce point size below 12;
  one space between sentences; smaller first-line indents; enable hyphenation; tighten line spacing
  off double.
- Replace Times New Roman with Equity; remove heading underlines; align headings flush with
  paragraph text; make headings only **half a point** larger than body; add space before/after.
- **Collaboration constraint:** if others will edit the memo in Word, prefer **system fonts** so it
  renders consistently on every machine; use numbered paragraphs for unambiguous cross-references;
  distribute as PDF (with commenting) when visual consistency matters.

## Letterhead

Principle: the letter text is **foreground**, the address block is **background** — typography must
keep the foreground dominant (the common failure is a giant address block overshadowing the letter).
- Reduce the top margin; left-align the address and signature blocks; narrow the line length /
  widen L-R margins; separate address lines with hard line breaks.
- Body in a serif (Equity); set the **address block in a sans** (e.g. Advocate) to distinguish it;
  use space-between-paragraphs instead of first-line indents; optional thin header/footer rules to
  unify.
- **Production tiers:** letterpress (best, priciest) → offset (mid; hire an independent designer) →
  internet offset printers (best for runs <500) → laser (cheapest). For laser letterhead: use
  professional fonts, buy premium off-white/ivory smooth wove paper, and print the name/address in a
  **pale color** (not black) so it reads as distinct from the grayish toner of the body text. When
  hiring a designer: choose carefully, then don't micromanage; never ask for free/spec work.

## Business cards

- One professional readable typeface (Equity); no novelty fonts. Remove anything nonessential.
- Build the layout **outward from the text**, not inward from the card edges — avoid the
  "baseball-diamond" layout that flings info into the four corners.
- Avoid: inconsistent/oversized name size; missing letterspacing on caps; ornamentation/pomposity.
- Substantial paper stock with good tactile feel; never laser-printed perforated card sheets
  ("flimsy and sad"); professional printing. Color understated — "the louder the color, the less of
  it you can use."

## Résumés

- Prioritize substance over cramming — most résumés can run **two pages**; ignore the one-page myth.
- Replace the system font (Calibri) with **Equity + Concourse**; increase margins; reduce line
  length; make headings smaller and lighter; **enlarge school and employer names** (they establish
  credibility by association); soften bullet styling; push nonessential detail to page two.
- Use good business letterhead paper (nothing more elaborate); submit digitally as **PDF**; print a
  two-page résumé on **separate sheets**, not duplex, so page two isn't missed.

## Grids of numbers

Typography must mirror the math: **in any column, digits of the same meaning align vertically.**
- Align by type: invoice numbers right; currency and weights on **decimal tabs**; zip codes left.
- Commas become mandatory in columns for alignment ("$6,736" must align under "$16,736" even though
  a standalone 6736 could drop the comma).
- Replace the system font (Calibri) with Concourse; remove needless color and borders; keep line
  spacing even. Format column labels only *after* the numbers align; slightly inconsistent label
  alignment is fine if it balances better.

## Presentations

Design for the room it's shown in — dim, large-scale — not your laptop.
- Start with a **black background** and a **thin sans-serif** at ~**50% gray** (not pure white); for
  daylight rooms use pale-gray backgrounds with medium-gray type; avoid harsh high-contrast combos.
- Pick a base size that fits **12–15 lines** per slide; keep sizes consistent across slides (turn
  off autofit/auto-resize); increase line spacing vs. print; avoid centered text; don't feel obliged
  to fill the screen width; color sparingly and pale; maximize white (or black) space.

## Contracts

Readers look for exits from dense text and mostly don't read contracts closely — design for that.
- Use **columns** for a comfortable line length; increase line spacing; add space between paragraphs
  with logical indenting; maximize white space for structure.
- **Body: 10-pt Equity** (larger would be better, but single-page-length pressure often forces 10
  pt); **headlines: Concourse**; eliminate ALL CAPS; bold/italic only for genuine emphasis.
- **Debunk the caps myth:** ALL CAPS does *not* reliably satisfy "conspicuous" requirements — courts
  judge conspicuousness contextually, and the Ninth Circuit has noted lawyers overestimate all-caps.
  A contract is "irreducibly complex"; the goal is making the required elements as effective as
  possible, not adding new ones.

## Court opinions

- Avoid monospaced fonts and character-based divider lines; use a readable body font; bold/italic
  sparingly; **small caps instead of underlining**; real **rules/borders** to separate sections.
- Wider margins; increased line spacing; centered text to organize structure; justified with
  hyphenation on.
- **Model — U.S. Supreme Court:** a **Century**-family font, justified with hyphenation, generous
  margins, unobtrusive headers, sections marked by white space rather than bold/italic. Note the
  binding rule: under **Supreme Court Rule 33, Times New Roman is forbidden** and a Century-family
  font is required. "Careful text-block design lets white space self-organize the page."
